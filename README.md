# EpubReader

В качестве основного архитектурного подхода был выбран MVVM (Model - View - ViewModel).

Представление построено на Jetpack Compose: оно отображает состояние и передаёт в модель
представления события пользователя. `ReaderViewModel` концентрирует состояние приложения (загрузка,
ошибка, режим чтения) и правила перехода между этими режимами. Доступ к содержимому книги и к
сохранённой позиции выполняется через контракты доменного слоя и репозитории - в терминах MVVM это
соответствует модели в широком смысле.

Такой выбор согласуется с задачей: доминируют сценарии вокруг одной книги и ограниченного набора
действий (навигация по главам, оглавление, сохранение прогресса), что естественно описывается
конечным автоматом. Сохранение логики при смене конфигурации экрана опирается на ViewModel и
механизмы платформы, без переноса бизнес-правил во встроенный компонент отображения веб-контента.
Раздельное сопровождение парсинга EPUB, персистентности и UI облегчает проверку и развитие модулей
по отдельности.

## 1. Архитектура

### 1.1 Общий подход

Проект собран в одном модуле `app`, при этом границы слоёв соблюдены: интерфейс не обращается к
файловой системе и парсеру напрямую; подготовка книги и сохранение позиции выполняются репозиториями
по интерфейсам домена. Зависимости между компонентами задаются через Dagger, без привлечения
закрытых сервисов.

### 1.2 Разделение на слои и зоны ответственности

| Слой                                         | Содержимое                                                                                                       | Ответственность                                                                                           |
|----------------------------------------------|------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------|
| **Интерфейс** (`ui`)                         | Экран чтения на Jetpack Compose, WebView через мост, панели, жесты по краям экрана, нижний лист оглавления       | Отображение состояния, обработка ввода, учёт жизненного цикла (в т.ч. сохранение позиции при уходе в фон) |
| **Модель представления** (`ReaderViewModel`) | Один поток состояния: загрузка, ошибка, готовность                                                               | Вызовы репозиториев, URL главы и фрагмента, восстановление прокрутки скриптом в странице                  |
| **Домен** (`domain`)                         | Модели данных (`PreparedBook`, `TocEntry`, `ReadingPosition`), интерфейсы репозиториев                           | Контракты без привязки к Android                                                                          |
| **Данные** (`data`)                          | Копирование и распаковка EPUB, разбор через epublib, позиция в DataStore (пара «ключ - значение»), пути к файлам | I/O, настройки, сопоставление оглавления с файлами spine                                                  |
| **Внедрение зависимостей** (`di`)            | Dagger, фабрика ViewModel                                                                                        | Сборка графа при старте процесса                                                                          |

Параметры отображения веб-контента вынесены в `ReaderWebViewSetup` (согласование масштаба с шириной
экрана, стабилизация вёрстки) отдельно от логики подготовки книги.

### 1.3 Управление состоянием

Состояние чтения описывается тремя вариантами: загрузка, ошибка, готовность. Оно публикуется через
`StateFlow`, на который подписан слой представления.

В режиме готовности в состоянии фиксируются: идентификатор книги, упорядоченный список файлов глав (
spine), текущий индекс, URL для загрузки во встроенный браузер, оглавление и при необходимости доля
вертикальной прокрутки для восстановления после запуска.

При переходе к другой главе или при выборе пункта оглавления модель представления пересчитывает URL
и обновляет ожидаемую прокрутку.

### 1.4 Обработка ошибок и нестандартные сценарии

Подготовка книги оформлена так, чтобы сбой (повреждённый ZIP, отсутствие файла, ошибка ввода-вывода,
отказ в доступе к архиву) не завершал работу без сообщения пользователю: отображается текст ошибки и
действие «повторить».

Если готовое состояние применить нельзя (например отсутствует файл главы на диске), экран
переводится в состояние ошибки.

Сбой записи позиции в хранилище не блокирует чтение; пользователю показывается краткое уведомление.

Если метаданные EPUB не содержат пригодного оглавления, формируется синтетический список по порядку
spine («Глава 1», «Глава 2» и т.д.).

Сетевой доступ не используется; контент поступает только из локального хранилища приложения.

### 1.5 Обоснование ключевых решений, ограничения, уязвимые места

| Решение                              | Зачем так                                                                   | Ограничения и риски                                                                                                                                                  |
|--------------------------------------|-----------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| WebView для HTML глав                | Отрисовка HTML, CSS и вложений по `file://` без собственного движка вёрстки | Зависимость от версии Chromium; тяжёлые документы и изображения; расхождения вёрстки между устройствами; инъекция стилей меняет исходное оформление                  |
| epublib                              | Разбор пакета и spine без полного стека Readium                             | Неполная поддержка отдельных расширений EPUB 3; сопровождение библиотеки на стороне сообщества                                                                       |
| Позиция как «глава + доля прокрутки» | Простое хранение и восстановление                                           | Нет привязки к абзацу или CFI; смена шрифта или вёрстки смещает место; при аварийном завершении процесса без события паузы последнее сохранение может не выполниться |
| DataStore для позиции                | Асинхронная запись без блокировки UI                                        | Схема без версионирования - при изменении полей требуется миграция                                                                                                   |
| Dagger 2                             | Явный граф зависимостей, ошибки видны при компиляции                        | Объём сгенерированного кода; при расширении графа нужна дисциплина в модулях; кодогенерация подключена через KSP (см. раздел 4)                                      |

Безопасность: предполагается работа только с распакованными файлами из собственного хранилища;
загрузка произвольных сетевых URL в тот же WebView без политики безопасности не предусмотрена.
Целостность архива EPUB криптографически не проверяется (достаточно для демонстрационного сценария;
для недоверенного контента нужны отдельные меры).

## 2. Эксплуатация в production

### 2.1 Поддержка: мониторинг, инциденты, реагирование

В эксплуатации ориентируются на консоль распространения приложения: отчёты о сбоях и ANR,
распределение по версиям и устройствам. При необходимости подключают внешний сервис группировки
стеков (например Crashlytics); в текущей реализации такая интеграция не выполняется.

Инцидентом считают, в частности, рост доли сбоев на новой версии, повторяющиеся стеки на одной
версии ОС или модели устройства, массовые ANR на одном сценарии.

Приоритет: сначала устранение блокирующих сбоев при открытии книги, затем регрессии после релиза,
затем второстепенные дефекты.

Реагирование: воспроизведение на той же сборке и том же EPUB, по возможности на той же версии
системы; анализ журнала (`adb logcat`); исправление в отдельной ветке; проверка на устройстве;
выпуск патча по принятому процессу релиза.

### 2.2 Обработка сбоев и ошибок

Источники: автоматические отчёты из консоли или стороннего сервиса, а также обращения с описанием
шагов воспроизведения.

Анализ: сопоставление стека с версией кода по тегу или номеру сборки; при сбоях во встроенном
компоненте отображения веб-страниц - проверка версии системного WebView на устройстве.

Исправление: задача с воспроизведением, изменение кода, регрессионная проверка сценариев чтения и
сохранения позиции, публикация сборки.

### 2.3 Релизы: выкладка и снижение рисков

Рекомендуется внутреннее или закрытое тестирование, затем публикация в магазин с поэтапным
распространением (staged rollout) и наблюдением метрик на первых долях аудитории.

Перед каждой публикацией в `app` поднимают `versionCode` и `versionName`.

Для учебной сборки релиз может подписываться так же, как debug. Для публичного магазина лучше
отдельный ключ, который не хранится в git.

### 2.4 Работа с EPUB: локальное хранение и ограничения

Демонстрационный файл размещается в `assets`, при первом запуске копируется во внутреннее хранилище
приложения, архив ZIP распаковывается в выделенный каталог; при уже существующей распаковке
повторный полный цикл не выполняется (условия задаёт репозиторий).

Структура книги считывается с диска из файла EPUB; порядок глав и оглавление извлекаются из
метаданных пакета.

Ограничения: дублирование места на диске (копия архива и распаковка); риск некорректного
сопоставления сохранённой позиции при замене файла книги при неизменном идентификаторе в метаданных;
не поддерживаются сценарии фиксированной вёрстки и расширенного EPUB 3 уровня издательских ридеров;
инъекция мета-тега области просмотра и стилей изменяет поведение относительно исходного XHTML без
доработки.

### 2.5 Производительность: риски и оптимизация

| Риск                                       | Что можно сделать                                                                                                                                                       |
|--------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Долго распаковывается большой архив        | Уже не в UI-потоке; при желании - прогресс или кэш распаковки по хешу файла                                                                                             |
| Очень тяжёлый HTML или крупные изображения | Ограничение на стороне контента; ленивая загрузка изображений в объёме проекта не реализована                                                                           |
| Лишние пересоздания WebView                | В текущей реализации одна долгоживущая страница на экран чтения; при развитии следует избегать повторной загрузки того же URL без необходимости                         |
| Память                                     | В отображении одна глава; при смене главы загружается новый документ; крупные вложения могут повысить потребление памяти - при необходимости использовать профилировщик |

## 3. Требования к сборке и окружению

### 3.1 Инструменты и версии

Сборка проверялась в стандартной среде: JDK требуемой версии, Android SDK с платформой и
инструментами сборки, доступ к репозиториям Maven (Google, Maven Central). Проприетарные плагины не
используются; для отладочной сборки закрытые ключи не требуются.

| Компонент                              | Версия (смотреть в репозитории)                    |
|----------------------------------------|----------------------------------------------------|
| Kotlin                                 | 2.3.20 (`gradle/libs.versions.toml`)               |
| Плагин Android для Gradle              | 9.0.1                                              |
| Gradle (wrapper)                       | 9.4.1 (`gradle/wrapper/gradle-wrapper.properties`) |
| JVM                                    | 17 (`jvmToolchain(17)` в `app/build.gradle.kts`)   |
| compileSdk / targetSdk                 | 36                                                 |
| minSdk                                 | 26                                                 |
| KSP (плагин `com.google.devtools.ksp`) | 2.3.6                                              |
| Версии библиотек                       | `gradle/libs.versions.toml`                        |

Рекомендуется актуальная версия Android Studio, совместимая с указанным плагином Android для Gradle.

### 3.2 Шаги сборки и запуска без изменений кода

1. Клонировать репозиторий.
2. Поставить JDK 17 и Android SDK (платформа API 36, инструменты сборки).
3. Открыть корень проекта в IDE или работать из корня в терминале.
4. Выполнить синхронизацию Gradle и загрузку зависимостей (при первом запуске требуется сеть).
5. Собрать debug:

```bash
./gradlew :app:assembleDebug
```

6. Поставить APK на устройство или эмулятор через IDE или `adb`.

Релиз:

```bash
./gradlew :app:assembleRelease
```

`local.properties` с путём к SDK в репозиторий не кладут; IDE обычно создаёт его сама при первой
настройке.

### 3.3 Артефакт APK

- Debug: `app/build/outputs/apk/debug/app-debug.apk`
- Release: `app/build/outputs/apk/release/app-release.apk`

APK, собранный из чистого состояния ветки без посторонних правок, соответствует коду при тех же
подписи и зависимостях.

### 3.4 Доступность инструментов

Зависимости разрешаются из репозиториев, перечисленных в `settings.gradle.kts`. Закрытые реестры
артефактов не требуются. Сетевой доступ необходим при первичной загрузке дистрибутива Gradle и
зависимостей при пустом локальном кэше.

## 4. Сторонние библиотеки: перечень, обоснование, ограничения

Точные версии зафиксированы в `gradle/libs.versions.toml` и в блоке `dependencies` в
`app/build.gradle.kts`.

| Зависимость                                 | Зачем                                           | Почему выбрали                                                        | Минусы                                                                                                                      |
|---------------------------------------------|-------------------------------------------------|-----------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------|
| **AndroidX Core KTX**                       | Удобства для Kotlin поверх Android              | Обычный выбор для Kotlin-проектов                                     | Привязка к версии SDK                                                                                                       |
| **Splash Screen (AndroidX)**                | Заставка при старте                             | Официальный способ под Android 12+                                    | Нужен согласованный стиль темы                                                                                              |
| **Lifecycle (runtime, ViewModel, Compose)** | Жизненный цикл, ViewModel, подписка из Compose  | Стандарт для экранов с состоянием                                     | Следить за правилами главного потока                                                                                        |
| **Activity Compose**                        | Встраивание Compose в активность                | Точка входа для Compose                                               | -                                                                                                                           |
| **Navigation Compose**                      | Переходы между экранами                         | Минимальный граф (один экран чтения)                                  | Версия согласована с остальным стеком                                                                                       |
| **Compose (BOM + UI + Material 3)**         | Интерфейс                                       | Версии через BOM одним блоком                                         | Размер APK, команде нужно знать Compose                                                                                     |
| **DataStore Preferences**                   | Сохранить позицию чтения                        | Современная замена старым SharedPreferences на запись                 | Схему ключей не версионируем - при смене полей нужна миграция                                                               |
| **Kotlin Coroutines (Android)**             | Асинхронность в репозитории и ViewModel         | Принятый в экосистеме Android подход                                  | Требуется явная обработка ошибок в построителях сопрограмм                                                                  |
| **epublib-core**                            | Разбор EPUB, spine, метаданные                  | Зрелая библиотека без полного стека Readium                           | LGPL; редкие варианты EPUB 3 могут разбираться неполностью; часть транзитивных зависимостей исключена в конфигурации модуля |

| **KSP** (плагин Gradle)                     | Обработка аннотаций на этапе сборки вместо kapt | Быстрее инкрементальные сборки, рекомендуемый путь для новых проектов |                                                                                                                             |

Компилятор Dagger подключён через конфигурацию `ksp`, не `kapt`. В `app/build.gradle.kts` у epublib
исключены транзитивные `slf4j-simple` (чтобы не конфликтовал с `slf4j-nop`) и `xmlpull` (чтобы не
дублировать классы с системным XML на Android).
